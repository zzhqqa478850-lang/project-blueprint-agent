# Project Blueprint - Skill 设计文档

## 1. 概念与愿景

**Project Blueprint** 是一款面向开发者的智能项目结构分析工具，旨在解决"项目文件一多就忘记哪个文件夹干什么用"的痛点。通过自动扫描项目目录、分析代码文件类型和内容，为每个文件夹生成清晰的功能说明，最终输出一份结构化的项目蓝图文档。

**核心理念**：像一本项目的"说明书"，让开发者能够快速理解任何一个陌生项目的结构和功能。

## 2. 核心功能

### 2.1 目录智能扫描

**功能描述**：递归扫描指定目录，生成完整的项目结构树

**关键特性**：
- 智能过滤：自动跳过 `node_modules`、`__pycache__`、`.git`、`dist`、`build` 等无需分析的目录
- 文件类型识别：通过文件扩展名判断类型（`.py` → Python，`.js` → JavaScript，`.md` → Markdown 等）
- 深度控制：可配置扫描深度，默认分析前 3 层目录

**支持的项目类型**：
- Python 项目（识别 `requirements.txt`、`pyproject.toml`、`setup.py`）
- JavaScript/TypeScript 项目（识别 `package.json`、`tsconfig.json`）
- Java 项目（识别 `pom.xml`、`build.gradle`）
- Go 项目（识别 `go.mod`）
- 通用项目（无特定配置文件）

### 2.2 项目类型检测

**功能描述**：通过关键配置文件自动识别项目类型和技术栈

**检测规则**：
```
配置文件 → 项目类型
requirements.txt / pyproject.toml → Python 项目
package.json → JavaScript/TypeScript 项目
pom.xml → Java Maven 项目
build.gradle → Java Gradle 项目
go.mod → Go 项目
Gemfile → Ruby 项目
composer.json → PHP 项目
```

**技术栈提取**：
- 从 `package.json` 提取 dependencies 和 devDependencies
- 从 `requirements.txt` 提取主要依赖
- 从配置文件中识别框架（React、Vue、Django、FastAPI 等）

### 2.3 智能说明生成

**功能描述**：为每个文件夹生成准确的功能描述

**分析策略**：

**策略一：配置文件分析**
- 读取文件夹内的配置文件
- 从配置中推断功能（如 `config.py` → 配置管理）

**策略二：入口文件分析**
- 识别入口文件（`__init__.py`、`main.py`、`index.js`、`App.jsx`）
- 分析入口文件了解模块职责

**策略三：文件命名模式**
- `utils/` → 工具函数库
- `controllers/` → 控制器/处理层
- `models/` → 数据模型
- `views/` → 视图/UI组件
- `services/` → 业务逻辑服务
- `middleware/` → 中间件
- `tests/` → 测试代码
- `docs/` → 文档资料
- `config/` → 配置文件

**策略四：内容摘要**
- 读取文件头部注释和文档字符串
- 提取关键函数/类名
- 分析文件内容生成摘要

### 2.4 文档生成

**功能描述**：将分析结果输出为美观的 Markdown 文档

**输出文件名**：`PROJECT_BLUEPRINT.md`（保存在项目根目录）

**文档结构**：


## 3. 技术实现

### 3.1 技术栈
- **语言**：Python 3.8+
- **核心库**：
  - `pathlib`：跨平台路径处理
  - `json`：配置文件解析
  - `re`：正则表达式匹配
  - `datetime`：时间戳处理

### 3.2 模块划分

```
project_blueprint/
├── __init__.py
├── scanner.py          # 目录扫描器
├── analyzer.py         # 项目分析器
├── generator.py        # 说明生成器
├── documenter.py       # 文档生成器
├── constants.py        # 常量定义（文件类型映射、过滤规则）
└── main.py             # 主入口
```

### 3.3 核心算法

**目录扫描算法**：
```
1. 接收根目录路径
2. 初始化空的结果列表
3. 递归遍历目录：
   a. 检查是否在过滤列表中 → 跳过
   b. 检查是否为文件 → 添加到结果
   c. 检查是否为目录 → 递归处理
4. 返回结构化的目录树
```

**说明生成算法**：
```
1. 遍历目录树中的每个文件夹
2. 获取该文件夹下的文件列表
3. 应用分析策略：
   a. 检查是否有配置文件
   b. 查找入口文件
   c. 应用命名模式规则
   d. 读取文件内容生成摘要
4. 综合分析结果生成描述
5. 返回 {路径: 说明} 字典
```

## 4. 用户交互

### 4.1 使用方式

**方式一：命令行调用**
```bash
python project_blueprint.py [项目路径]
```

**方式二：集成到 Trae Skill**
```
用户输入：帮我分析这个项目的结构
Skill 响应：正在扫描项目目录...
```

### 4.2 配置选项

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `path` | 当前目录 | 要分析的项目路径 |
| `depth` | 3 | 最大扫描深度 |
| `output` | PROJECT_BLUEPRINT.md | 输出文件名 |
| `exclude` | [默认过滤列表] | 额外需要排除的目录 |

## 9. 风险与挑战

### 9.1 可能的困难
- **大项目性能**：深度嵌套的项目可能导致扫描时间过长
- **说明准确性**：自动生成的说明可能不够准确，需要用户审核
- **多语言混合**：一个项目中使用多种语言时，分析策略需要适配

### 9.2 应对方案
- 添加深度限制和超时机制
- 提供用户修正接口，允许手动修改生成的说明
- 分别处理不同语言的文件，再合并结果
